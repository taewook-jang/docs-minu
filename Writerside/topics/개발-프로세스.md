# 개발 프로세스

## Git 활용
- 브랜치 전략
    - feature/
    - develop
    - main
- 커밋 메시지
  ```
  feat: 새로운 기능 추가
  fix: 버그 수정
  docs: 문서 수정
  ```

## 코드 리뷰
- 리뷰 요청하기
    - PR(Pull Request) 작성법
    - 리뷰어 지정
- 피드백 반영
    - 코드 수정
    - 재검토 요청

## 배포 프로세스
- 환경별 배포
    - 개발(dev)
    - 스테이징(stage)
    - 운영(prod)
- CI/CD
    - Jenkins 파이프라인
    - 자동 배포 설정
